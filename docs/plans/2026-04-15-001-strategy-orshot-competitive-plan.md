# Orshot competitive attack plan

## Context

Orshot.com generates **~6× Pictify's organic search traffic** (ETV 1,371/mo vs 228/mo) from a portfolio of template-named tool and template pages targeting consumer-creator intent — certificates, Canva alternatives, tweet screenshots, word-definition images. Pictify ranks for HTML→image / URL→image developer terms; the two sites share essentially zero SERPs (**1 intersecting keyword** as of baseline — "create fake link to image", SV 40).

The opportunity: **35,850 SV/mo of gap keywords** where Orshot ranks top 30 (SV ≥ 50) and Pictify ranks nowhere. The top 13 keywords (SV ≥ 500 each) alone account for 27,610 SV/mo — 77% of the total gap. Most have KD 0–30, so ranking is realistic without heavy link building.

**Our wedge:** build orshot-parity tools and content, but position each one under Pictify's "Programmable Image Engine" brand — API-first where orshot is no-code-only. Each tool page becomes an ad for the API via a visible, edited-state-accurate code snippet that converts dev-ICP visitors.

## Positioning hooks

Every new page/cluster should ladder up to these:

1. **"Programmable Image Engine"** — not "creative automation." Pictify's differentiators: real expression engine, conditional logic, data bindings, AI copilot, GIF/video roadmap.
2. **API + no-code duality** — orshot's tools are no-code only. Every Pictify tool page has a visible, edited-state-accurate code snippet that doubles as a sales demo.
3. **Primary buyer:** SaaS engineering/product leads. Even on consumer-intent keywords like "certificate generator," the page pitches the API for automation.

## Prioritization rule

When considering any new tool/blog page: **"does orshot rank for it, and is KD ≤ 30?"** If yes, it's a candidate. If no, deprioritize regardless of SV.
